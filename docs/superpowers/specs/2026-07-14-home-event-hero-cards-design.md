# 营销首页 · 活动大图卡对齐与英雄区同壳

> 日期：2026-07-14  
> 状态：已确认  
> 范围：预览站营销首页「精选活动与赛事」+「英雄广场」横滑卡  
> 参考：效果图「精选活动与赛事」全幅背景活动卡

---

## 1. 目标

按效果图统一首页两类卡片的视觉壳：

1. **精选活动与赛事**：微调现有 `event-card--hero`，对齐效果图像素级观感。
2. **英雄广场横滑**：改用同一套大图卡壳；内容改为该英雄的一条代表赛事/活动，不再展示教练头像/荣誉/三行链接。

---

## 2. 卡片视觉结构（两区共用）

复用并微调 `event-card--hero` 作为统一卡壳：

| 层级 | 元素 | 规格 |
|------|------|------|
| 背景 | 全幅封面图 | `object-fit: cover`，圆角约 12px |
| 遮罩 | 底部加深渐变 | 保证白字可读 |
| 左上 | 时间胶囊 | 半透明黑底 + 色点（赛事黄 / 活动橙）+ `MM/DD (周X) HH:mm–HH:mm` |
| 左中 | 类型标签 | 「赛事」黄 / 「活动」橙 |
| 左下信息 | 标题 + 地点 | 白字；标题约 16px 加粗，地点约 12px |
| 右下 | 价格 + CTA | `¥N/人` + 白底圆角「立即报名」 |

视觉微调重点（相对现状）：

- 「立即报名」更接近效果图的圆角白钮（可略增圆角）
- 时间胶囊、类型标签、底栏价格与按钮的间距/对齐与效果图一致
- 遮罩强度保证标题与价格清晰可读

---

## 3. 区块行为

### 3.1 精选活动与赛事

- 纵向列表展示活动大图卡（现结构保留）。
- 数据：公开招募中 `type = event | activity`，展示前 3 条；静态兜底可与现稿一致。
- 字段：封面 / 起止时间 / 类型 / 标题 / 地点 / 费用 → 卡壳槽位。
- 整卡点击 → `recruitment-detail.html?id=…`。
- 「查看更多」保持现状（可占位）。

### 3.2 英雄广场横滑

- 仍为横向滚动；单卡宽约 280px，高度与活动卡一致（约 148–160px）。
- 数据源：英雄列表；每名英雄取 **1 条**代表招募：优先 `event`，否则 `activity`；都没有则跳过该英雄。
- 卡面只展示该条招募信息（无头像、昵称、荣誉、课程行）。
- 去重：若与下方「精选」前几条 `recruit_id` 相同，优先换该英雄的下一条可用招募；仍冲突则允许保留。
- 整卡点击 → 对应招募详情（不再进英雄详情）。
- 「查看更多」→ `heroes.html`（英雄 Tab）不变。

## 5. 边界与兜底

| 情况 | 处理 |
|------|------|
| API 不可用 | 保留静态兜底 HTML（已改为新卡壳） |
| 费用缺失 | 显示「价格待定」，仍可进详情 |
| 封面缺失 | 默认 `event.jpg` 或 `recruit-cover.jpg` |
| 时间缺失 | 时间胶囊隐藏或显示「时间待定」 |
| 英雄无赛事/活动 | 不进入横滑；若横滑为空则隐藏整块「英雄广场」区块 |

## 6. 验收要点

1. 「精选活动与赛事」三张卡：时间胶囊、类型标签、标题、地点、价格、「立即报名」与效果图一致。
2. 「英雄广场」横滑卡与活动卡同壳；无头像/荣誉/三行链接。
3. 点英雄区卡 → 招募详情；点「查看更多」→ 英雄列表。
4. 无代表赛事/活动的英雄不出现在横滑。
5. `营销首页.md` 描述与实现一致。
