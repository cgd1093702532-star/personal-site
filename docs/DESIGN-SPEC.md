# 个人站设计规范说明

> **参考站点**：[唐韧的「AI个体户」计划](https://tangren.org.cn/)  
> **文档版本**：v1.0  
> **更新日期**：2026-06-30  
> **适用范围**：个人品牌 + 服务/内容转化型单页网站

---

## 1. 设计定位

### 1.1 站点类型

- **形态**：单页滚动落地页（One-page Landing）
- **目标**：建立信任 → 展示能力与方法论 → 引导咨询/合作/订阅
- **气质**：专业可信、温暖克制、偏知识型个人品牌，避免过度炫技

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **结果优先** | 首屏传达「你能获得什么」，而非「我是谁」的简历式罗列 |
| **数字背书** | 用可量化指标（粉丝、年限、项目数、学员数等）建立信任 |
| **编号体系** | 统一使用 `01 / 02 / 03` 编号，强化方法论与结构感 |
| **节奏分明** | 白底区块与浅蓝 band 区块交替，避免视觉疲劳 |
| **转化清晰** | 顶栏、首屏、定价区、页脚均保留明确 CTA |
| **移动优先** | 所有 Grid 布局需在窄屏下优雅折叠为单列 |

### 1.3 内容漏斗（信息结构）

```
承诺（Hero）
  ↓
信任（关于我 + 数据 + 时间轴）
  ↓
共鸣（痛点 / 场景四宫格）
  ↓
方案（产品 / 服务 + 交付大纲）
  ↓
价值打包（你能获得什么）
  ↓
筛选（适合谁 / 不适合谁）
  ↓
证明（案例 + 统计数据）
  ↓
决策（价格 / 合作方式 + FAQ）
  ↓
行动（联系 / 扫码 / 预约）
```

## 2. 色彩系统

### 2.1 Design Tokens

```css
:root {
  /* 品牌色 */
  --brand: #004080;           /* 主品牌蓝：主按钮、链接、强调文字 */
  --brand-deep: #052b52;      /* 深蓝：大标题、深色卡片背景 */
  --brand-soft: #e7f0f9;      /* 浅蓝：标签底、编号圆底、头像边框区 */

  /* 文字色 */
  --ink: #17212b;             /* 正文主色 */
  --muted: #66727f;           /* 次要说明、卡片描述 */
  --nav-text: #31475f;        /* 导航链接 */

  /* 背景色 */
  --paper: #fbf7ef;           /* 页面主底色：暖米纸感 */
  --band: #f4f8fc;            /* 区块 band 背景：浅灰蓝 */
  --white: #ffffff;

  /* 暖色强调 */
  --warm: #c89954;            /* 金色：编号、时间轴高亮、引用左边线 */
  --warm-soft: #f4eadc;       /* 暖色浅底：warm panel 渐变 */

  /* 边框与阴影 */
  --line: rgba(0, 64, 128, 0.14);
  --shadow: 0 24px 70px rgba(15, 38, 70, 0.14);
  --shadow-sm: 0 14px 34px rgba(15, 38, 70, 0.07);
  --shadow-btn: 0 16px 34px rgba(0, 64, 128, 0.24);

  /* 圆角 */
  --radius: 8px;
  --radius-pill: 999px;
}
```

### 2.2 色彩使用规则

| 用途 | 颜色 | 禁止 |
|------|------|------|
| 页面背景 | `--paper` + 顶部浅蓝渐变 | 纯白大面积铺底（显冷、缺层次） |
| 一级标题 | `--brand-deep` | 用 `--brand` 或 `--warm` 做正文标题 |
| 正文 | `--ink` | 纯黑 `#000` |
| 说明文字 | `--muted` | 过浅导致对比度不足 |
| 编号 / 序号 | `--warm` | 与正文同色，失去层级 |
| 主 CTA 按钮 | `--brand` 底 + `--white` 字 | 金色做大面积按钮底 |
| 深色强调卡 | `--brand-deep` 底 + `--white` 字 | 多块深色卡连续堆叠 |
| 区块分隔 band | `--band` | 与 `--paper` 无差异 |

### 2.3 背景渐变

**页面全局背景：**

```css
background:
  linear-gradient(180deg, rgba(231, 240, 249, 0.74), rgba(251, 247, 239, 0.3) 620px),
  var(--paper);
```

**卡片 / 信息图背景：**

```css
background:
  linear-gradient(135deg, rgba(255, 255, 255, 0.92), rgba(231, 240, 249, 0.72));
```

**暖色 Panel 头：**

```css
background: linear-gradient(180deg, #fffaf2, #ffffff);
```

## 3. 字体与排版

### 3.1 字体栈

```css
font-family:
  -apple-system, BlinkMacSystemFont, "Segoe UI",
  "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
  sans-serif;
```

不使用装饰性字体；中文以系统字体保证加载速度与可读性。

### 3.2 字号层级

| 元素 | 字号 | 行高 | 字重 | 颜色 |
|------|------|------|------|------|
| H1（Hero 主标题） | `clamp(34px, 4vw, 56px)` | 1.05 | 默认 | `--brand-deep` |
| H2（区块标题） | `clamp(30px, 4vw, 50px)` | 1.16 | 默认 | `--brand-deep` |
| H3（卡片标题） | `20px` ~ `21px` | 1.35 | 默认 | `--brand-deep` |
| 区块副标题 | `clamp(24px, 2.6vw, 36px)` | 1.25 | 默认 | `--brand-deep` |
| Hero 价值主张 | `clamp(16px, 1.5vw, 19px)` | 1.6 | 700 | `--brand` |
| 正文 | `16px` | 1.7 | 400 | `--ink` |
| 卡片描述 | `16px` | 1.7 | 400 | `--muted` |
| 导航链接 | `14px` | — | 400 | `--nav-text` |
| 编号标签 | `14px` | — | 900 | `--warm` |
| 统计数字 | `clamp(24px, 2.4vw, 34px)` | 1.18 | 900 | `--brand` 或 `--warm` |
| 按钮文字 | `15px` | — | 800 | 依按钮类型 |

### 3.3 排版规则

- 标题 `letter-spacing: 0`，不额外加字间距
- Hero 主标题允许 `<br />` 手动断行，控制视觉节奏
- 价值主张（promise）使用 **左侧 3px 金色竖线** + 左内边距 `14px`
- 区块标题与正文之间留白 `20px`；区块之间纵向间距 `72px` ~ `100px`
- 中文文案避免超过 28 字不换行的长句；卡片描述控制在 2~3 行

## 4. 布局与间距

### 4.1 全局容器

```css
/* 标准内容区 */
width: min(1180px, calc(100% - 40px));
margin: 0 auto;

/* 全宽 band 区块（delivery / gains） */
width: 100%;
padding: 100px max(20px, calc((100% - 1180px) / 2));
```

### 4.2 间距尺度

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-xs` | `8px` | 紧凑元素间距 |
| `--space-sm` | `14px` | 按钮组 gap、列表项 gap |
| `--space-md` | `18px` ~ `22px` | 卡片 grid gap |
| `--space-lg` | `26px` ~ `34px` | 卡片内边距 |
| `--space-xl` | `56px` | Hero 双栏 gap |
| `--space-2xl` | `72px` ~ `100px` | 区块上下 padding |

### 4.3 栅格布局

| 模块 | 桌面布局 | 平板 / 手机 |
|------|----------|-------------|
| Hero | `grid: 1.03fr / 0.97fr` 双栏 | 单列，视觉区置后或折叠 |
| 痛点四宫格 | `repeat(4, 1fr)` | `repeat(2, 1fr)` → 单列 |
| 产品双 Panel | `repeat(2, 1fr)` | 单列 |
| 课程大纲 | `repeat(2, 1fr)` | 单列 |
| 权益区 | `0.88fr / 1.12fr` 左主卡右列表 | 单列，主卡在上 |
| 统计数字 | `repeat(4, 1fr)` | `repeat(2, 1fr)` |
| 案例卡片 | 自适应 Grid，`minmax(280px, 1fr)` | 单列 |

### 4.4 页面结构（区块顺序）

| 序号 | 区块 ID | 区块名称 | 背景 |
|------|---------|----------|------|
| 0 | `#top` | Sticky 顶栏 | 半透明 `--paper` + blur |
| 1 | — | Hero 首屏 | `--paper` 渐变 |
| 2 | `#about` | 关于我 + 时间轴 | `--paper` |
| 3 | `#method` | 痛点四宫格 | `--paper` |
| 4 | `#delivery` | 产品 / 服务交付 | `--band` |
| 5 | `#gains` | 你能获得什么 | `--band` |
| 6 | — | 谁适合 | `--paper` |
| 7 | `#cases` | 成功案例 | `--paper` |
| 8 | `#pricing` | 定价 / 合作方式 | `--paper` |
| 9 | — | FAQ | `--paper` |
| 10 | `#apply` | 底部 CTA + 联系 | `--paper` 或深色 |

## 5. 组件规范

### 5.1 顶栏（Site Header）

- **高度**：`72px`
- **定位**：`sticky; top: 0; z-index: 20`
- **背景**：`rgba(251, 247, 239, 0.86)` + `backdrop-filter: blur(18px)`
- **底边框**：`1px solid rgba(0, 64, 128, 0.1)`

**结构：**

```
[圆形头像 36px] [品牌名]          [导航链接 × 2] [胶囊 CTA]
```

- 品牌名：`17px / 700 / --brand-deep`
- 导航链接：`14px`，hover 变 `--brand`
- 导航 CTA：描边胶囊，`border: 1px solid --brand`，无填充

### 5.2 按钮（Button）

| 类型 | 样式 |
|------|------|
| **Primary** | 背景 `--brand`，字 `--white`，圆角 `--radius-pill`，min-height `50px`，阴影 `--shadow-btn` |
| **Secondary** | 背景 `rgba(255,255,255,0.65)`，边框 `--line`，字 `--brand-deep` |
| **Nav CTA** | 同 Secondary 但更紧凑，min-height `40px` |
| **Full** | `width: 100%` |
| **Text Link** | 无背景，字 `--brand`，hover 下划线 |

**交互：**

```css
.button:hover {
  transform: translateY(-2px);
  transition: transform 180ms ease, box-shadow 180ms ease, background 180ms ease;
}
```

### 5.3 卡片（Card）

**标准卡片：**

```css
border: 1px solid var(--line);
border-radius: var(--radius);
background: rgba(255, 255, 255, 0.72);
padding: 26px; /* 或 22px~34px 依场景 */
```

**深色主卡（Gain Primary / Definition Card）：**

```css
background: var(--brand-deep);
color: var(--white);
box-shadow: var(--shadow);
padding: 34px;
```

**编号卡片（痛点 / 权益列表项）：**

- 左上角或左侧：`01` 编号，`14px / 900 / --warm`
- 标题下接 `--muted` 描述

### 5.4 Hero 信息图（Monetization Map）

右侧视觉区，展示「能力 / 服务闭环」：

```
┌─────────────────────────────────┐
│  [01 模块A]  [02 模块B]         │
│  [03 模块C]  [04 模块D]         │
│─────────────────────────────────│
│  底层能力条（如 AI 工作流）      │
└─────────────────────────────────┘
```

- 外层：`border-radius: 8px`，渐变白蓝背景
- 四格 flow：每格含编号 + 标题 + 一行描述
- 底部 base 条：强调底层支撑能力

### 5.5 关于我（About Card）

```
┌──────────┬────────────────────────────┐
│  头像     │  姓名 H3                    │
│  圆形     │  一句话定位                  │
│          │  2 段简介                   │
│          │  [统计1] [统计2] [统计3] [统计4] │
└──────────┴────────────────────────────┘
```

- 头像：圆形，`object-fit: cover`
- 统计格：上数字（`--brand` / 900），下标签（`--muted` / `14px`）

### 5.6 时间轴（Timeline）

- 中央竖线 + 左右交替排列事件
- 每项结构：`年份` · `圆点` · `事件名` · `描述`
- 普通项：圆点 `--brand`
- **高亮项**（当前阶段）：圆点 / 年份 / 事件名均用 `--warm`
- 竖线末端：金色三角箭头

### 5.7 产品 Panel（Delivery Panel）

双栏卡片，结构：

```
[编号 01]                    [编号 02]
[Panel 标题 H3]              [Panel 标题 H3]
[描述段落]                   [描述段落]
• 要点 1                     • 要点 1
• 要点 2                     • 要点 2
```

- 列表项前缀：金色圆点 `8px`
- `.warm` 变体：暖色渐变头部 `#fffaf2 → #fff`

### 5.8 课程 / 大纲列表（Curriculum Day）

```
┌─ Day Header（深蓝底或暖色渐变）────────┐
│  Day 标签 + 标题 + 副标题              │
├──────────────────────────────────────┤
│  01  课程条目 1                       │
│  02  课程条目 2                       │
│  ...                                 │
└──────────────────────────────────────┘
```

- 深色 header：`--brand-deep` 底 + 白字
- 暖色 header：`.curriculum-day.warm` 渐变
- 列表条目：编号 `--warm` + 条目名

### 5.9 权益区（Gains Layout）

左右分栏 `0.88 : 1.12`：

- **左侧主卡**：深蓝底，核心交付一句话 + 3 个指标（如「1 份 / 1 个 / 1 套」）
- **右侧列表**：4 条，每条左侧圆形编号底 `--brand-soft`

### 5.10 案例区（Cases）

- 顶部 4 个统计数字横排
- Tab 筛选：全部 / 分类1 / 分类2 / 分类3
- 卡片网格：封面图 + 平台标签 + 标题 + 成果描述
- 点击卡片：Lightbox 弹窗展示详情

### 5.11 定价卡（Price Card）

- 居中布局
- 价格数字突出（大字号 + `--brand-deep`）
- 权益 bullet 列表
- 底部 Primary 全宽按钮

### 5.12 FAQ

使用原生 `<details>` / `<summary>` 手风琴，无需 JS：

```html
<details>
  <summary>问题标题</summary>
  <p>回答内容</p>
</details>
```

### 5.13 底部 CTA（Apply）

- 强文案 H2（结果导向，可含换行）
- 微信二维码或联系方式
- 辅助说明（如「添加时备注：xxx」）

## 6. 动效与交互

### 6.1 滚动

```css
html { scroll-behavior: smooth; }
```

锚点跳转用于顶栏导航与 CTA 按钮。

### 6.2 Hero 入场动画

```css
@keyframes hero-rise {
  from { opacity: 0; transform: translateY(16px); }
  to   { opacity: 1; transform: translateY(0); }
}
```

- 时长：`620ms`
- 缓动：`cubic-bezier(0.22, 0.61, 0.36, 1)`
- 子元素依次延迟 `0.04s` 递增

### 6.3 无障碍

```css
@media (prefers-reduced-motion: reduce) {
  /* 禁用 hero-rise 等动画 */
}
```

### 6.4 悬停反馈

| 元素 | 反馈 |
|------|------|
| 按钮 | `translateY(-2px)` + 阴影变化 |
| 导航链接 | 颜色变 `--brand` |
| 案例卡片 | 轻微上浮或阴影加深（可选） |
| Text Link | 下划线 |

## 7. 文案规范

### 7.1 标题写法

| 类型 | 公式 | 示例 |
|------|------|------|
| Hero 主标题 | 结果 + 情感共鸣 | 「一个人也能做一家公司」 |
| 区块标题 | 痛点 / 价值 / 行动 | 「如果你也遇到以下问题」 |
| 卡片标题 | 具体场景 | 「想学 AI，不懂从哪开始」 |
| CTA | 动词 + 低门槛 | 「立刻咨询报名」「我想报名咨询」 |

### 7.2 数据指标

- 使用「数字 + 单位 + 标签」三段式：`16万+` / `全网粉丝`
- 数字用 `+` 表示「以上」，增加感知规模
- 每处统计不超过 4 个，避免信息过载

### 7.3 编号体系

全站统一两位数编号：`01` `02` `03` …  
用于：痛点卡、产品 Panel、课程条目、权益列表、受众分类。

### 7.4 语气

- 第二人称「你」拉近距离
- 具体场景优于抽象形容词
- 避免「资深」「顶尖」等空泛自封

## 8. 响应式断点

| 断点 | 宽度 | 调整策略 |
|------|------|----------|
| Desktop | `≥ 1024px` | 完整多栏布局 |
| Tablet | `768px ~ 1023px` | 四宫格变两列，Hero 可保持双栏或折为单列 |
| Mobile | `< 768px` | 全部单列；顶栏导航可收为汉堡菜单；字号用 `clamp` 自适应 |

**移动端必查：**

- [ ] 顶栏 CTA 不被挤压消失
- [ ] 按钮组可换行，触控区域 ≥ 44px
- [ ] 时间轴在窄屏改为左对齐单列
- [ ] 二维码居中且宽度 ≥ 160px
- [ ] 图片与卡片无横向溢出

## 10. 无障碍与 SEO

### 10.1 无障碍

- 所有图片提供有意义的 `alt`
- 导航使用 `<nav aria-label="主导航">`
- 按钮与链接文字表意清晰，避免纯「点击这里」
- 颜色对比度：正文 `--ink` on `--paper` ≥ 4.5:1
- FAQ 使用语义化 `<details>` 元素
- 尊重 `prefers-reduced-motion`

### 10.2 SEO

```html
<title>你的名字 · 定位关键词</title>
<meta name="description" content="一句话价值主张，120~160 字。" />
<html lang="zh-CN">
```

- 每个区块用语义化 `<section>` + `id` 锚点
- 案例页可后续扩展为独立 URL 以提升收录
