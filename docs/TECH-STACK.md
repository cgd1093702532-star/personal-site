> **文档版本**：v1.0  
> **更新日期**：2026-07-01  
> **已确认决策**：首版纯 HTML + CSS · GitHub Pages 部署 · 首版不含访问统计  
> **关联文档**：[DESIGN-SPEC.md](./DESIGN-SPEC.md) · [TECH-SPEC.md](./TECH-SPEC.md) · [FRONTEND-CONSTRAINTS.md](./FRONTEND-CONSTRAINTS.md)

## 1. 项目背景

| 维度 | 决策 |
|------|------|
| 站点形态 | 单页滚动落地页（One-page Landing） |
| 核心目标 | 建立信任 → 展示能力 → 引导咨询/合作 |
| 内容语言 | 简体中文（`lang="zh-CN"`） |
| 维护者 | 单人维护，优先低运维、低学习成本 |
| 首版策略 | 快速上线，避免过度工程化 |

**首版不做：**

- 用户登录 / 后台 CMS
- 访问统计（Umami、Plausible、百度统计等）
- 复杂动画与 3D 效果
- 服务端动态内容
- 多语言

## 2. 首版技术方案（已确定）

### 2.1 一句话说明

首版用 **纯 HTML + CSS** 做一页网站，放到 **GitHub Pages** 上，访客打开就能看，**不加访问统计**。

### 2.2 技术组成

| 层级 | 选型 | 说明 |
|------|------|------|
| 页面结构 | 语义化 HTML5 | 浏览器直接可读；利于 SEO；无需构建工具 |
| 样式 | 原生 CSS + Design Tokens | 颜色、间距等规则见 [DESIGN-SPEC.md](./DESIGN-SPEC.md) §2 |
| 脚本 | 原生 JavaScript（可选） | 仅用于 Tab 切换、移动端菜单等轻交互；也可零 JS |
| 图片 | WebP（必要时 PNG 备用） | 头像 < 50KB，案例图按需压缩 |
| 字体 | 系统字体栈 | 不加载外部字体，国内打开更快 |
| 部署 | **GitHub Pages** | 免费静态托管，与 Git 仓库绑定 |
| 访问统计 | **首版不含** | 上线稳定后如需再加（见 §4） |

### 2.3 为什么选这套方案

| 考量 | 说明 |
|------|------|
| 简单 | 非技术人员也能理解「一个 HTML 文件 + 样式文件 + 图片文件夹」 |
| 快 | 无 npm 安装、无构建步骤，改完刷新浏览器即可预览 |
| 稳 | GitHub Pages 免费、可靠，适合个人品牌站 |
| 轻 | 页面加载快，符合 DESIGN-SPEC 性能目标（LCP < 2.5s） |
| 够用 | 单页转化型站点，首版功能完全覆盖 |

## 3. 部署：GitHub Pages

### 3.1 是什么

GitHub Pages 是 GitHub 提供的 **免费网页托管服务**。把网站文件放进 GitHub 仓库并开启 Pages 后，即可通过网址访问。

### 3.2 访问地址形式

| 类型 | 示例 |
|------|------|
| 默认地址 | `https://{用户名}.github.io/{仓库名}/` |
| 自定义域名（可选，后续配置） | `https://你的域名.com` |

### 3.3 发布流程（概要）

1. 在 GitHub 创建仓库，上传网站文件（`index.html` 等在仓库根目录）
2. 仓库 Settings → Pages → Source 选择分支（通常 `main`）
3. 等待 1~3 分钟，访问生成的网址
4. （可选）在域名服务商处添加 CNAME，绑定自定义域名

详细验收标准见 [TECH-SPEC.md](./TECH-SPEC.md) §6。

## 4. 后续扩展路线（非首版）

当站点需要 **博客、案例详情页、内容频繁更新** 时，再考虑升级。首版无需实施。

| 阶段 | 触发条件 | 可能方案 |
|------|----------|----------|
| Phase 2 | 需要 3+ 独立子页面或文章列表 | Astro 静态站点生成器 |
| Phase 2+ | 组件复用增多 | Tailwind CSS + TypeScript |
| 可选 | 上线后关心访问量 | Umami / Plausible 等轻量统计 |
| 可选 | 需要在线表单 | Formspree / 飞书 webhook |

**迁移原则：** 保留现有 URL 结构与 section 锚点，避免影响已分享的链接和 SEO。

## 5. 开发工具（建议）

| 工具 | 用途 | 是否必须 |
|------|------|----------|
| Cursor / VS Code | 编辑代码 | 推荐 |
| 浏览器（Chrome / Safari） | 预览与调试 | 必须 |
| **打开网站预览.command** | 本地一键预览（推荐，不依赖 localhost） | 必须 |
| Live Server 插件 | 可选，非必须 | 可选 |
| Git + GitHub | 版本管理与部署 | 必须 |
| Prettier | 代码格式化 | 可选 |
| Lighthouse（Chrome 内置） | 性能与无障碍检测 | 推荐，上线前跑一次 |

**首版不需要：** Node.js 项目、npm 依赖、React/Vue 等框架。

## 6. 关键决策记录

### ADR-001：首版采用纯 HTML + CSS

- **状态**：已确认  
- **理由**：单页、交互少；与 DESIGN-SPEC 一致；非技术人员维护成本最低  
- **何时重审**：出现博客、多页面或频繁内容更新需求时

### ADR-002：部署平台选 GitHub Pages

- **状态**：已确认  
- **理由**：免费、与 Git 工作流一体、适合静态站点  
- **备注**：后续可绑定自定义域名，无需换平台

### ADR-003：首版不含访问统计

- **状态**：已确认  
- **理由**：减少首版配置复杂度；不影响核心转化功能（联系、扫码）  
- **何时重审**：网站稳定运行 1~2 个月后，若需要了解访问量再添加

### ADR-004：CSS 变量作为唯一设计 token 源

- **状态**：已确认  
- **理由**：DESIGN-SPEC 已定义完整 token 体系  
- **禁止**：在代码中硬编码色值（如 `#004080`），应写 `var(--brand)`

### ADR-005：不引入 Google Fonts

- **状态**：已确认  
- **理由**：国内访问不稳定；系统字体栈已满足品牌气质
