## 1. 页面业务目标

「我的报名」是用户查看**自己报过名的活动与课程**的列表页。

主要解决两件事：

1. **按状态浏览**：进行中 / 已结束两个 Tab（按活动结束时间拆分）
2. **回到活动详情**：招募类报名可点卡片进入 [招募详情](./招募详情.md)

---

## 2. 登录和身份描述

| 身份 | 用户大概情况 | 页面上多出来 / 不一样的地方 |
|------|--------------|------------------------------|
| 全部登录用户 | 有个人中心「我的报名」等入口 | 双 Tab 列表 |
| 无报名记录 | 从未报名 | 对应 Tab 空态 |

> 未登录用户若从个人中心进入，应先走授权（以实现为准）。

### 2.1 进行中

展示尚未结束的活动报名。卡片带「进行中」标签；展示时间、地点、费用 · 报名状态 · 支付状态。

### 2.2 已结束

展示已结束的活动。卡片带「已结束」标签；其余字段类似。

---

## 3. 页面详细描述

导航栏标题：**我的报名**。

### 3.1 顶部 Tab

| Tab | 说明 |
|-----|------|
| 进行中(n) | 括号内数量 |
| 已结束(n) | 括号内数量 |

### 3.2 报名卡片

| 展示 | 说明 |
|------|------|
| 活动名称 | |
| 进行中 / 已结束标签 | |
| 📅 时间 | |
| 📍 地点 | 有则显示 |
| ¥费用 · 报名状态 · 支付状态 | 单行摘要，如「¥299 · 已报名 · 已支付」 |

**点击卡片：** 招募类报名可进招募详情；纯课程报名当前可能无法跳转。

### 3.3 空态

| Tab | 图标 | 标题 | 提示 |
|-----|------|------|------|
| 进行中 | 📋 | 暂无进行中的报名 | 去首页看看热门招募，报名参加精彩水上活动 |
| 已结束 | ✅ | 暂无已结束的报名 | 已结束的活动会显示在这里，方便查看历史记录 |

---

## 4. 常见路径

- **查报名：** 我的 → 我的报名 → 切换 Tab  
- **进详情：** 点招募类报名卡片 → [招募详情](./招募详情.md)  
- **报名后回看：** [招募详情](./招募详情.md) 报名成功 → 间接刷新本页数据

## 6. 规则补充与验收要点
