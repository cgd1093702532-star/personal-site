# 英雄管理（管理后台）

> 状态：草稿 · P0 · M1 · 平台：Web 管理后台

---

## 1. 页面概述

| 项 | 值 |
|---|---|
| 路由 | `/admin/heroes` · `/admin/heroes/:id` |
| 权限 | 管理员（本地不鉴权） |
| 源码 | `admin/src/pages/Heroes.vue` |

## 2. 英雄列表

- **模式：** 申请审核 / 已认证英雄
- **申请 Tab：** 待审核 / 已通过 / 已驳回
- **筛选：** 场景、申请时间范围（规划中）
- **搜索：** 姓名、手机号
- **表格：** 头像、姓名、手机号、项目类型、申请时间、状态
- **操作：** 查看、批准、驳回、删除申请

## 3. 英雄详情审核

- 基本信息、数据统计、简介
- 过往荣誉（含认证状态）
- 资质证书（点击放大）
- **批准：** `POST /api/admin/applications/{id}/approve`（通过后创建/关联英雄）
- **驳回：** `POST /api/admin/applications/{id}/reject`（须填原因）
- **删除：** `DELETE /api/admin/applications/{id}`（可连带删除已关联英雄）

## 4. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/applications` | 申请列表（可按 status 筛选） |
| `GET /api/admin/applications/{id}` | 申请详情 |
| `POST /api/admin/applications/{id}/approve` | 批准（仅 pending） |
| `POST /api/admin/applications/{id}/reject` | 驳回（仅 pending，写原因） |
| `DELETE /api/admin/applications/{id}` | 删除申请（可连带删英雄） |
| `GET /api/admin/heroes` | 已认证英雄列表 |

> 实现路径为 **applications**，非文档旧稿中的 `/api/admin/heroes/{id}/approve|reject`。

## 5. 正常流程

待审列表 → 查看详情 → 批准 / 驳回 / 删除。

## 6. 业务规则

1. Tab：待审 / 已通过 / 已驳回；另有已认证英雄列表。
2. 仅 `pending` 状态可批准或驳回。
3. 批准成功后创建（或关联）英雄记录。
4. 驳回须写入驳回原因。
5. 删除申请时可连带删除已关联英雄。
6. 本地管理端不鉴权。

## 7. 异常与边界

- 对非 `pending` 申请执行批/驳 → `invalid_status`。
- 页面 layout 注入后，脚本须能正确绑定批准/驳回/删除按钮（预览与 Vue 页均适用）。
