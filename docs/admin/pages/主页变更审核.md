# 主页变更审核（管理后台）

> 状态：草稿 · P2 · M3 · 平台：Web 管理后台

---

## 1. 功能

- **Tab：** 待审核 / 已通过 / 已驳回
- **列表：** 英雄、变更类型、提交时间
- **对比视图：** 修改前 / 修改后，差异高亮
- **预警：** 异常变更（如大面积删除荣誉）— 见待确认
- **操作：** 通过、驳回、批量删除

## 2. 页面结构

1. Tab 列表
2. before / after 对比详情
3. 通过 / 驳回操作
4. 勾选 + 批量删除

## 3. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/profile-changes?status=&page=&page_size=` | 变更申请列表 |
| `GET /api/admin/profile-changes/{change_id}` | 变更详情（含 before/after） |
| `POST /api/admin/profile-changes/{change_id}/approve` | 通过（仅 pending） |
| `POST /api/admin/profile-changes/{change_id}/reject` | 驳回（仅 pending） |
| `POST /api/admin/profile-changes/batch-delete` | 批量删除（body：`{ "ids": ["change_id", ...] }`） |

## 4. 说明

通过后将 `after` 合并写入 `heroes`；仅 `pending` 可操作。

## 5. 正常流程

待审列表 → 查看 before/after → 通过或驳回。

## 6. 业务规则

1. Tab：待审 / 通过 / 驳回。
2. 仅 `pending` 可批准或驳回。
3. 通过时将 `after` 合并到 `heroes`。
4. 详情须展示 before / after 对比。
5. 支持勾选后批量删除变更记录（各状态均可删；删除后不可恢复）。

## 7. 异常与边界

- 非 `pending` 操作应拒绝（如 `invalid_status`）。
- 合并冲突或英雄不存在时需明确错误提示。
- 批量删除未勾选时提示先勾选。
