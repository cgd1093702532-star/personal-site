## 1. 页面业务目标

「我的评价」展示与用户相关的**评价列表**（当前为演示只读列表）。

主要解决一件事：

- **回顾评价记录**：评价者头像、昵称、星级、分数、正文与时间

个人中心「已评价 N」展示评价数量。当前版本**无**在本页发表或编辑评价的入口。

> 产品语义待统一：本页与 [英雄评价列表](./英雄评价列表.md)（教练收到的评价）数据边界见 §6。

---

## 2. 登录和身份描述

| 身份 | 用户大概情况 | 页面上多出来 / 不一样的地方 |
|------|--------------|------------------------------|
| 全部用户 | 个人中心「我的评价」 | 评价列表或空态 |
| 无评价 | 尚未有记录 | **暂无评价** |

> 无额外权限门槛；有数据与无数据仅差列表 / 空态。

---

## 3. 页面详细描述

导航栏标题：**我的评价**。

### 3.1 评价卡片

布局与英雄评价列表相同：

| 展示 | 说明 |
|------|------|
| 头像 | 评价者头像或占位 |
| 昵称 | 如「学员小李」 |
| 星级 + 分数 | 支持半星 |
| 评价正文 | 有则展示 |
| 时间 | 如 2026-06-15 |

按时间从新到旧排列。点击当前无反应。

### 3.2 空态

| 文案 |
|------|
| **暂无评价** |

## 6. 规则补充与验收要点

### 6.1 已对齐（产品已确认可验收）

- 展示评价卡片：头像、昵称、星级（含半星）、分数、正文、时间
- 按时间降序排列
- 无评价时展示「暂无评价」
- 仅展示平台允许对外可见的评价
- 当前版本只读，无发表 / 编辑入口

### 6.2 还没做完

- 页面语义待确认：展示「我写的评价」还是「与我相关的评价」
- 谁可以评价、是否须先完成活动，待产品规则
- 发表 / 编辑评价入口
- 卡片上关联活动名称展示
