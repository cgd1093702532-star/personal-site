## 1. 页面概述

| 项 | 值 |
|---|---|
| 路由 | `/admin/heroes`（预览 `heroes.html`） |
| 新建/编辑 | `supplier-edit.html` / `supplier-edit.html?id=` |
| 权限 | 管理员（本地不鉴权） |
| 菜单位置 | 供需管理 → 供方列表 |

## 2. 列表页结构

1. **筛选区**（白底卡片，标签在上，对齐 UI 规范 tokens）
   - 供方 id、供方昵称/手机号码、项目类型、供方状态、审核状态
   - 操作：查询、重置、导出、查看已生成订单报表
2. **操作栏**：+ 创建供方（主按钮）、批量审核（描边主色）
3. **Tab**：全部 / 申请入驻待审核 (n) / 资料变更待审核 (n)
4. **表格列**：勾选、供方ID、供方头像和昵称、手机号（脱敏）、身份证号（脱敏）、项目类型、资质等级、申请渠道、入驻审核状态、审核人员、审核时间、创建时间、操作（查看 / 通过 / 驳回 / 启用·禁用 / 删除）
5. **分页**：每页 10 条；切换 Tab / 查询 / 重置时回到第 1 页

## 3. 操作

| 操作 | 行为 |
|------|------|
| 创建供方 | 跳转新建页（单列表单 + 底栏） |
| 详情（已入驻） | 跳转编辑页（操作文案「查看」） |
| 详情（待审申请） | 打开审核弹窗（操作文案「查看」） |
| 通过（待审申请） | 打开审核弹窗，默认选中「通过」 |
| 驳回（待审申请） | 打开审核弹窗，默认选中「驳回」并填写原因 |
| 启用 / 禁用 | 切换已入驻供方是否可用（`enabled`）；禁用时需填写原因 |
| 删除 | 确认后删除供方或入驻申请；删除供方时连带删除关联申请，并回写小程序认证状态（无其余申请则为未认证） |
| 批量审核 | 对勾选的待审申请打开审核 |

## 4. 新建 / 编辑页（UI 规范）

单列纵向表单，标签在上，必填星号在文案后；分区标题使用灰底通栏（参考活动表单样式）；底栏右对齐固定。

### 4.1 分区与字段

| 分区 | 字段 | 必填 | 控件 |
|------|------|------|------|
| 基本信息 | 供方昵称 | 是 | Input，最多 20 字 |
| 基本信息 | 供方姓名 | 是 | Input，单行文本 |
| 基本信息 | 供方头像 | 是 | Upload×1，建议 800×800 |
| 基本信息 | 手机号码 | 是 | Input |
| 基本信息 | 身份证号 | 是 | Input |
| 基本信息 | 收货地址 | 否 | Input，单行文本 |
| 基本信息 | 银行账户 | 否 | Input，单行文本 |
| 资质认证 | 项目类型 | 是 | 下拉选择；「新增」打开「管理项目类型」弹框（列表按创建时间倒序，可编辑/删除；「新增项目类型」再叠一层新建弹框）；「刷新」从管理列表拉取最新选项（按创建时间倒序） |
| 资质认证 | 资质等级 | 是 | Select |
| 资质认证 | 从业年限 | 否 | Select：1-3 年 / 3-5 年 / 5-10 年 / 10 年+ |
| 资质认证 | 荣誉成就 | 否 | Input + 添加，最多 10 个；标签 10 字以内 |

底栏：取消（描边）+ 创建/保存（主色实心）

## 5. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/heroes` | 供方列表 |
| `POST /api/admin/heroes` | 创建供方 |
| `PUT /api/heroes/{id}` | 更新供方 |
| `GET /api/admin/applications` | 入驻申请列表 |
| `POST /api/admin/applications/{id}/approve\|reject` | 审核申请 |
| `GET /api/admin/profile-changes?status=pending` | 资料变更待审计数 |

## 6. 业务规则

1. 列表合并已入驻供方与待审/驳回申请；资料变更 Tab 依赖 profile-changes pending。
2. 后台创建渠道记为「后台创建」，审核状态默认「通过」。
3. 仅 `pending` 申请可审核。
4. 表单校验失败时高亮首个错误项并滚动定位。
5. 删除已入驻供方：删除 `heroes` 记录及 `hero_id` 关联的 `hero_applications`，并按该用户最新申请回写 `mock_hero_role`；若已无申请则置为未认证（空/`none`）。
6. 删除入驻申请：可连带删除已关联英雄，并同样回写认证状态。
