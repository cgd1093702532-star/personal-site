## 2. 业务需求

### 2.1 业务目标

- 按招募**标题**过滤 `my_signups` 中非课程报名，展示报名人姓名、手机、时间、状态
- 状态 **待确认** 时显示「确认」按钮 → 更新为 **已确认**
- 页头展示招募标题

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 发布者 | ✅（从 my-recruitments 进入） |
| 普通用户 | M1 无入口守卫 |

### 2.3 核心业务规则

1. `options.title` decodeURIComponent，默认 **招募报名**
2. filter：`type !== 'course'` 且 `item.title === title`（精确匹配）
3. onShow 刷新列表
4. onConfirm → `updateSignupStatus(id, '已确认')`

## 3. 页面结构与 UI 元素规格

### 3.1 信息架构

```
.sub-page
├── .sub-page__head（招募标题）
└── .sub-page__item × N
    ├── title: name · phone
    ├── meta: time · status
    └── btn「确认」（status=待确认）
```

### 3.2 UI 元素清单

| 元素 ID | 类型 | 文案/格式 | 数据来源 | 交互 |
|---------|------|-----------|----------|------|
| head | 文本 | recruitTitle | options.title | 无 |
| item-title | 文本 | **{{name}} · {{phone}}** | list | 无 |
| item-meta | 文本 | **{{time}} · {{status}}** | list | 无 |
| btn-confirm | 按钮 | **确认** | status===**待确认** | onConfirm |
| empty | 隐式 | 无 item 时不渲染列表项 | filter 空 | — |

> M1 无专用空态文案，filter 为空则仅显示 head。

### 4.2 列表项字段
