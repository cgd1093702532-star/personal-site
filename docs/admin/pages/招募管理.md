# 招募管理（管理后台）

> 状态：草稿 · P0 · M1 · 平台：Web 管理后台

---

## 1. 列表

- **Tab：** 全部 / 招募中 / 已满员 / 已结束 / 已取消
- **筛选：** 场景、类型、发布时间
- **搜索：** 标题
- **表格：** 标题、场景、英雄、报名情况、价格、时间、状态
- **操作：** 查看详情、关闭招募

## 2. 详情弹窗

- 基本信息、时间地点、进度、富文本详情
- 已报名人员：姓名、电话、时间、支付/报名状态
- **关闭：** `POST /api/admin/recruitments/{recruit_id}/close`
- **编辑招募** — 字段同小程序发布表单（是否开放见待确认）

## 3. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/recruitments` | 招募列表（可读公开招募数据） |
| `GET /api/admin/recruitments/{recruit_id}` | 招募详情 |
| `GET /api/admin/recruitments/{recruit_id}/signups` | 该招募下报名 |
| `POST /api/admin/recruitments/{recruit_id}/close` | 关闭招募 → `cancelled` |

## 4. 页面结构

1. Tab + 筛选/搜索
2. 列表表格
3. 详情弹窗（含报名人员）

## 5. 正常流程

打开列表 → 查看详情（含报名）→ 必要时关闭招募。

## 6. 业务规则

1. 后台读取公开招募数据（与前台同源 SQLite）。
2. 关闭招募将状态置为 `cancelled`。
3. 关闭后影响前台报名入口（不可再报名）。

## 7. 异常与边界

- 已关闭招募再次关闭应幂等或提示已取消。
- 关闭不影响历史报名记录的查询。
