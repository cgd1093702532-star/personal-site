## 2. 业务需求

### 2.1 业务目标

- 展示**教练收到**的学员评价列表，对应个人中心「评分」统计点击入口
- 与 [我的评价](./我的评价.md) 区分：本页 `getMyRatings()` / app-state `hero_ratings`
- M1 只读，无回复/举报

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 已认证英雄 | ✅（profile onMyRatings 守卫） |
| 普通用户 | ❌ 无入口 |

### 2.3 核心业务规则

1. `getMyRatings()` → enrichReview + 时间降序
2. UI 与 my-reviews 相同：头像、昵称、星、分、内容、时间
3. 空态文案：**暂无评分**（与我的评价「暂无评价」不同）
4. onLoad/onShow refresh

## 3. 页面结构与 UI 元素规格

### 3.1 信息架构

```
.my-review（同 my-reviews）
└── .my-review__item × N
```

### 3.2 UI 元素清单

| 元素 | 说明 | 数据来源 |
|------|------|----------|
| avatar | 评价者头像/占位 | reviewer_avatar |
| name | reviewer_nickname | 如 **学员A** |
| stars | ★ 5 颗半星支持 | enrichReview |
| score | 数字分 | score |
| content | 评价正文 | 可选 wx:if |
| time | timeDisplay | 可选 wx:if |
| empty | **暂无评分** | length=0 |
