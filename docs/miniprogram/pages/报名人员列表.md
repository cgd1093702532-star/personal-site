## 1. 页面业务目标

「报名人员列表」有两种用法：

1. **招募名单**：活动发布者查看某条招募的报名，并可确认「待确认」报名
2. **课程已报名成员**：教练从「我的课程」查看某门课的报名成员（只读）

---

## 2. 登录和身份描述

| 身份 | 用户大概情况 | 页面上看到什么 |
|------|--------------|----------------|
| 招募发布者 | 从我的招募「查看报名 / 查看名单」进入 | 与课程成员同版式：头像/昵称/手机/时间；另有状态；待确认可点「确认」 |
| 课程教练 | 从我的课程「已报名成员」进入 | 标题 + 头像/昵称/手机号/报名时间（倒序） |
| 无人报名 | 尚无人报名 | 空态文案 |
| 普通用户 | 无 C 端入口 | 不应进入 |

## 3. 页面详细描述

### 3.1 课程已报名成员（`course_id`）

导航栏标题：**已报名成员**。

| 展示 | 说明 |
|------|------|
| 页头 | 课程名称 |
| 成员行 | 头像、昵称、手机号、报名时间 |
| 排序 | 按报名时间**倒序**（新报名在上） |
| 时间格式 | `YYYY年M月D日 HH:mm:ss`（如 `2026年7月14日 18:32:15`） |

无确认/拒绝操作，只读。

### 3.2 招募报名名单（无 `course_id`）

导航栏标题：**报名人员**。

版式与 §3.1 课程「已报名成员」一致（头像 + 昵称 + 手机号 + 报名时间倒序，时间格式同 §3.1）。

| 内容 | 说明 |
|------|------|
| 页头 | 招募标题 |
| 成员行 | 头像、昵称、手机号、报名时间 |
| 状态 | 行内展示（如待确认 / 已确认） |
| 确认按钮 | 仅 **待确认** 时在行右侧显示「确认」 |

点「确认」→ 提示「已确认」→ 列表刷新为已确认。

## 6. 规则补充与验收要点

### 6.1 已对齐（产品已确认可验收）

- 课程模式：头像、昵称、手机号、报名时间（年月日 时分秒），按报名时间倒序
- 招募模式：与课程同版式；另有状态；待确认可确认
- 空态有文案
- 从其他页面返回时列表应刷新

### 6.2 还没做完

- 招募名单后续改为按招募唯一标识匹配（避免重名）
- 「拒绝报名」及完整状态流转
- 导出名单
