# 后台管理系统 · 新建 / 编辑页 UI 设计规范

> 适用范围：英雄广场 **Web 管理后台** 的新建、编辑、表单录入页  
> 参考来源：活动创建/编辑表单截图（2026-07-10）  
> 实现基线：`preview/assets/admin.css` 中的 admin tokens 与表单组件  
> 状态：已确认（视觉规范） · 最后更新：2026-07-10

---

## 1. 设计定位

| 项 | 值 |
|---|---|
| 页面类型 | 新建 / 编辑（表单页） |
| 视觉体系 | Ant Design 风格后台：白底、主色蓝、轻边框、弱圆角 |
| 布局模式 | **单列纵向表单**（标签在上、控件在下） |
| 内容区背景 | 页面灰底 `#f0f2f5`，表单卡片白底 `#ffffff` |
| 主操作区 | **底部固定操作栏**（始终可见） |
| 字体 | 系统无衬线（`-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`） |

本规范只约束「新建 / 编辑」类页面；列表、筛选、表格页另见列表页实践，可复用同一套 tokens。

## 2. Design Tokens

与现有 `admin.css` 对齐，新建/编辑页优先使用下列变量。

### 2.1 颜色

```css
:root {
  /* 品牌 / 交互 */
  --admin-primary: #1890ff;
  --admin-primary-hover: #40a9ff;
  --admin-primary-active: #096dd9;
  --admin-primary-bg: #e6f7ff;          /* 上传区、轻提示底 */

  /* 功能色 */
  --admin-danger: #ff4d4f;              /* 必填星号、错误 */
  --admin-success: #52c41a;
  --admin-warning: #faad14;

  /* 文字 */
  --admin-text: rgba(0, 0, 0, 0.85);           /* 输入内容、主文案 */
  --admin-text-label: #5b8def;                 /* 表单标签（参考图偏蓝灰） */
  --admin-text-secondary: rgba(0, 0, 0, 0.45); /* 辅助说明 */
  --admin-text-placeholder: #bfbfbf;           /* placeholder */

  /* 背景 */
  --admin-bg: #f0f2f5;                  /* 页面底 */
  --admin-bg-card: #ffffff;             /* 表单卡片 / 底栏 */
  --admin-bg-disabled: #f5f5f5;

  /* 边框 */
  --admin-border: #f0f0f0;              /* 卡片分割、底栏顶线 */
  --admin-border-input: #d9d9d9;        /* 输入框默认边框 */
  --admin-border-dashed: #91d5ff;       /* 上传虚线框 */
}
```

### 2.2 字号与字重

| Token | 字号 | 字重 | 行高 | 用途 |
|-------|------|------|------|------|
| 标签 | 14px | 400 / 500 | 22px | 表单项 label |
| 正文 / 输入 | 14px | 400 | 22px | input、select、textarea 内容 |
| 辅助说明 | 12px | 400 | 20px | 上传建议尺寸、字数限制旁注 |
| 按钮 | 14px | 400 | 1 | 底栏按钮文字 |
| 链接 | 14px | 400 | 22px | 「搜索地图」等行内操作 |

### 2.3 间距与尺寸

| Token | 值 | 用途 |
|-------|-----|------|
| 表单项垂直间距 | 24px | 相邻 form-item 之间 |
| 标签与控件间距 | 8px | label → control |
| 表单卡片内边距 | 24px | 内容区 padding |
| 输入框高度 | 32px | 单行 input / select / 日期条 |
| 输入框圆角 | 2px～4px | 与现网 admin 一致，取 **2px** |
| 上传方块 | 96×96px（可 80～104） | 封面上传 |
| 底栏高度 | 56px～64px | 含上下 padding |
| 底栏按钮间距 | 12px | 次要 / 主要按钮间距 |
| 表单内容最大宽度 | 640px～720px | 单列可读宽度；过宽时左对齐，勿拉满整屏 |

### 2.4 阴影与层级

| 元素 | 样式 |
|------|------|
| 表单卡片 | `0 1px 2px rgba(0,0,0,0.03), 0 1px 6px rgba(0,0,0,0.04)`（可与列表卡片一致） |
| 底栏 | 顶部分割线 `1px solid #f0f0f0`；可选轻微上阴影 `0 -2px 8px rgba(0,0,0,0.04)` |
| 输入 focus | `border-color: #1890ff` + `box-shadow: 0 0 0 2px rgba(24,144,255,0.2)` |

## 3. 页面结构

```
┌─────────────────────────────────────────────┐
│ 顶栏 / 面包屑（沿用后台壳）                    │
├──────────┬──────────────────────────────────┤
│ 侧栏     │  主内容区（灰底）                   │
│          │  ┌────────────────────────────┐  │
│          │  │ 白底表单卡片                 │  │
│          │  │  · 表单项（纵向堆叠）        │  │
│          │  │  · …                       │  │
│          │  └────────────────────────────┘  │
│          │  ┌────────────────────────────┐  │
│          │  │ 固定底栏：次要按钮 | 主按钮   │  │
│          │  └────────────────────────────┘  │
└──────────┴──────────────────────────────────┘
```

### 3.1 布局规则

1. **单列**：默认一列；仅当字段极短且成对出现（如经纬度）才允许两列。
2. **标签在上**：禁止「左标签右输入」的横向表单（与参考图一致）。
3. **必填标记**：红色 `*` 紧跟标签文案之后（如 `活动标题*`）；颜色 `#ff4d4f`。
4. **滚动**：表单主体可滚动；**底栏固定在主内容区底部**（相对 `admin-main`，不随表单滚出视口）。
5. **面包屑**：`模块 / 列表名 / 新建|编辑`，与现有后台一致。

## 4. 表单组件规范

### 4.1 通用表单项（Form Item）

```
[标签*]
[控件]
[可选：12px 辅助说明]
```

| 状态 | 表现 |
|------|------|
| 默认 | 边框 `#d9d9d9`，文字主色 |
| Hover | 边框 `#1890ff` |
| Focus | 边框 `#1890ff` + 外发光 |
| 禁用 | 背景 `#f5f5f5`，文字次要色，不可编辑 |
| 错误 | 边框 `#ff4d4f`；标签下或控件下显示 12px 红色错误文案 |

**Placeholder**：说明用途与限制（字数、格式），颜色 `#bfbfbf`。  
示例：`请输入活动标题，最多50个字`。

### 4.2 单行文本（Input）

- 高度 32px，左右 padding 11px
- 用于：标题、名称、地址、手机号等
- 有字数上限时：placeholder 写明；可选在右侧或下方显示 `已输入/上限`

### 4.3 多行文本（Textarea）

- 最小高度约 72px～96px
- 允许右下角拖拽调整高度（参考图）
- 用于：备注、说明
- Placeholder 示例：`可填写活动特殊说明`

### 4.4 下拉选择（Select）

- 外观与 Input 同高、同圆角
- 未选时显示占位文案（灰色），选中后为主文字色
- 右侧箭头图标

### 4.5 日期 / 时间范围（Date Range）

结构（单行容器）：

```
[ 开始日期 ]  →  [ 结束日期 ]                    📅
```

| 规则 | 说明 |
|------|------|
| 容器 | 高度 32px，整段一条边框，勿拆成两个独立框 |
| 分隔 | 中间箭头 `→` |
| 图标 | 右侧日历图标，次要色 |
| 占位 | 「开始日期」「结束日期」 |
| 必填 | 标签带 `*`（如活动时间） |

### 4.6 图片上传（Upload）

| 规则 | 说明 |
|------|------|
| 外观 | 正方形虚线框，边框色偏主色（`#91d5ff`），背景浅白/浅蓝 |
| 图标 | 居中 `+`，主色 `#1890ff` |
| 尺寸建议 | 默认 96×96；多图时横向排列，间距 8px～12px |
| 说明文案 | 框下方 12px 次要色，如：`建议尺寸800*800px，最多可上传1张` |
| 已上传 | 显示缩略图；hover 可删除 / 替换 |
| 必填 | 标签带 `*` |

### 4.7 带操作的输入（Input + Affix / Link）

用于地址、搜索类字段：

```
[ 请输入活动地址                          🔍 ]  搜索地图
```

| 元素 | 规范 |
|------|------|
| 输入框内图标 | 右侧放大镜，次要色 |
| 行外链接 | 主色文字按钮，如「搜索地图」，与输入框垂直居中，间距 8px～12px |
| 点击链接 | 打开地图选点或搜索面板，回填地址 |

### 4.8 标签输入（Tag Input）

- 外观同单行 Input
- Placeholder 写清规则：`输入后可添加，最多3个，自定义标签最多5字`
- 已添加标签：输入框内以 Tag 胶囊展示，可删除
- 超出上限：禁止继续添加并 Toast / 行内提示

### 4.9 富内容 / 图文入口

参考图「活动详情」：

- 必填标签 + 操作入口按钮（非大段富文本直接铺开）
- 按钮文案示例：`插入图文跳转`
- 样式：浅底或描边按钮，可带图标；点击进入图文编辑 / 跳转配置
- 已配置后：展示摘要卡片（标题、封面缩略图），可编辑 / 删除

### 4.10 开关 / 单选 / 多选

新建编辑页若出现：

- 开关：主色开启态
- Radio / Checkbox：主色选中，与 Ant Design 默认一致
- 选项文案 14px，与控件间距 8px

## 5. 底部操作栏

### 5.1 结构

```
┌──────────────────────────────────────────────┐
│  [保存暂不发布]    [发布]              (?)   │  ← 主按钮组居中或偏左；帮助可选
└──────────────────────────────────────────────┘
```

参考图为：**次要描边按钮 + 主色实心按钮**，水平排列，整体偏左/居中于内容区。

### 5.2 按钮层级

| 类型 | 样式 | 典型文案 |
|------|------|----------|
| 主按钮 Primary | 实心蓝底白字 `#1890ff` | 发布、提交、保存 |
| 次按钮 Outline | 白底、蓝边、蓝字 | 保存暂不发布、存草稿 |
| 危险按钮 | 红底或红字 | 删除（编辑页少用在底栏） |
| 文字按钮 | 无边框主色字 | 取消（若需要返回列表） |

### 5.3 交互规则

1. 主按钮在右，次要在左（符合「先草稿、后发布」阅读顺序）。
2. 提交中：主按钮 loading，禁止重复点击。
3. 校验失败：滚动到首个错误项并高亮。
4. 离开未保存：二次确认（可选）。
5. 帮助 `?`：可选；固定在视口右下角时不遮挡主按钮。

### 5.4 文案约定（后台通用）

| 场景 | 次要 | 主要 |
|------|------|------|
| 可发布内容（招募/课程/活动） | 保存暂不发布 | 发布 |
| 纯资料编辑 | 取消 | 保存 |
| 审核处理 | 取消 | 提交 |
| 创建供方等实体 | 取消 | 创建 |

## 6. 字段编排原则

1. **先核心后补充**：标题 / 封面 / 时间 / 地点 → 组织信息 → 标签 → 备注 → 详情。
2. **必填靠前**：必填字段集中在上半屏，降低漏填。
3. **一条字段一件事**：不要把互不相关的控件塞进同一行。
4. **限制写在 placeholder 或辅助文案**：字数、张数、尺寸、格式。
5. **危险操作不进底栏主区**：删除放页内次要位置或更多菜单。

## 7. 状态与反馈

| 场景 | 表现 |
|------|------|
| 加载详情（编辑） | 卡片内 Skeleton 或居中「加载中…」 |
| 保存成功 | Toast 成功 → 返回列表或停留并提示 |
| 发布成功 | Toast → 跳转列表或详情 |
| 网络失败 | Toast 错误，保留已填内容 |
| 空必填提交 | 字段错误态 + 首错滚动 |

---

## 8. 响应式

| 宽度 | 行为 |
|------|------|
| ≥ 1200px | 表单内容最大宽约 720px，左对齐于主内容区 |
| 768px～1199px | 表单宽 100%，卡片 padding 16px～24px |
| < 768px | 少见；底栏按钮可等分拉满，上传区保持方块 |

## 9. 组件与 CSS 类映射（实现指引）

落地时优先复用 / 扩展现有类名，避免另起一套：

| 表单卡片 | `.admin-card` + `.admin-form` |
| 表单项 | `.admin-form-item` |
| 标签 | `.admin-form-item__label` / `--required` |
| 输入 | `.admin-input` |
| 下拉 | `.admin-select` |
| 日期范围 | `.admin-date-range` |
| 上传 | `.admin-upload`（待补） |

> 注：当前 `admin-form-item__label--required` 使用 `::before` 星号；若严格对齐参考图「文案后星号」，可改为 `::after` 或标签后内联 `*`。

## 10. 参考字段模板（活动类）

以下为参考图字段结构，供招募/课程/活动等编辑页对照，不强制业务同名：

| 字段 | 必填 | 控件 | 说明文案示例 |
|------|------|------|----------------|
| 标题 | 是 | Input | 请输入活动标题，最多50个字 |
| 封面 | 是 | Upload×1 | 建议尺寸800*800px，最多可上传1张 |
| 时间 | 是 | Date Range | 开始日期 → 结束日期 |
| 地址 | 否 | Input + 搜索地图 | 请输入活动地址 |
| 主办方名称 | 否 | Input | 请输入主办方名称 |
| 标签 | 否 | Tag Input | 最多3个，自定义最多5字 |
| 备注 | 否 | Textarea | 可填写活动特殊说明 |
| 详情 | 是 | 图文入口 | 插入图文跳转 |
