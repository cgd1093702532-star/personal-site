## 2. 业务需求

### 2.1 业务目标

- 独立页上传资质证书图片（最多 **5** 张），写入 `hero_apply_certs` app-state
- **主流程**已在 [我的英雄资料](./我的英雄资料.md) 内联管理；本页供 [申请成为英雄](./申请成为英雄.md) 或预览链路使用

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 申请中/英雄 | ✅ |
| M1 无路由守卫 | — |

### 2.3 正常流程

选择证书图（最多 5 张）→ 保存写入申请/资料相关状态。

### 2.4 核心业务规则

1. 设计上敏感变更应走 `profile_change_requests`，通过后写 `heroes`
2. 最多 5 张；保存时至少 1 张
3. 主流程亦可在「我的英雄资料」内联管理；本页供申请/预览链路
4. M1 可能仍直写 app-state / `updateHero`

### 2.5 异常与边界

- 未选证书不可保存

### 2.6 待确认项

- [ ] 哪些字段必须审核 vs 可直写 `updateHero`

| 元素 ID | 类型 | 文案（精确） | 交互 |

---

## 4. 字段与校验矩阵

| 字段 key | 控件 | 必填 | 规则 | 错误 Toast | 存储 |
|----------|------|------|------|------------|------|
| certUrls | chooseMedia | ✅ 保存时 | 1–5 张图片 path | **请上传证书** | hero_apply_certs |
| certCount | 计算 | — | certUrls.length | — | — |

### 4.1 提交 payload

```json
{
  "value": ["wxfile://tmp1.jpg", "wxfile://tmp2.jpg"]
}
```

写入 `setAppState('hero_apply_certs', certUrls)`。
