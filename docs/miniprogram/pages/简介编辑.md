## 2. 业务需求

### 2.1 业务目标

- 编辑长文本简介/关于我，maxlength **500**
- 从 [我的英雄资料](./我的英雄资料.md) 进入：读 storage `hero-profile-edit-bio`，保存 `updateHero('1', { about_me })`
- 其他来源 M1 Mock 保存 Toast **保存成功（Mock）**

### 2.2 适用角色与权限

| 场景 | 可否进入 |
|------|----------|
| hero-profile 关于我 | ✅ 已认证英雄 |
| 其他 from | ✅ M1 无守卫 |

### 2.3 正常流程

进入 → 预填简介 → 校验 ≥10 字 → 保存（设计上敏感字段应走审核）。

### 2.4 核心业务规则

1. 设计上敏感变更应走 `profile_change_requests`，通过后写 `heroes`
2. 从英雄资料进入：保存 `about_me`（M1 可能直写 `updateHero`）
3. trim 后长度 <10 → Toast「内容至少10字」；maxlength 500

### 2.5 异常与边界

- API 失败时可写本地 storage 兜底（M1）

### 2.6 待确认项

- [ ] 哪些字段必须审核 vs 可直写 `updateHero`

## 4. 字段与校验矩阵

| 字段 key | 标签 | 控件 | 必填 | 规则 | 错误 Toast | API |
|----------|------|------|------|------|------------|-----|
| bio | 简介/关于我 | textarea | ✅ | trim 后 ≥10 字；≤500 | **内容至少10字** | about_me |

### 4.1 路由参数

| 参数 | 值 | 效果 |
|------|-----|------|
| from | hero-profile | 读写在 storage + updateHero |
| field | about | 与 from 组合触发关于我模式 |
| （无） | — | Mock 默认文案 + Mock 保存 |

### 4.2 Storage 键

| 键 | 方向 | 说明 |
|----|------|------|
| hero-profile-edit-bio | 进入时读 | 预填 |
| hero-profile-edit-bio-result | API 失败写 | 本地兜底 M1 |
