## 2. 业务需求

### 2.1 业务目标

- 展示用户报名的活动/课程：进行中 vs 已结束（按 `end_at` 拆分）
- 卡片展示时间、地点、费用、状态、支付状态
- 点击卡片 → [招募详情](./招募详情.md)（需 recruit_id；纯课程报名 M1 可能无 id）

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 全部用户 | ✅ |

### 2.3 核心业务规则

1. `getMySignupLists()` → active/ended + enrichSignup（补 timeDisplay/location/fee）
2. Badge：**进行中** / **已结束** 由 listTab 决定
3. Meta 行：`¥{fee} · {status} · {payStatus}` 示例 **¥500 · 已报名 · 待支付**
4. onItemTap：有 recruit_id 才 navigateTo recruitment-detail
5. 个人中心 summary：ongoing=active.length, done=ended.length

## 3. 页面结构与 UI 元素规格

### 3.1 信息架构

```
.my-signup__tabs（进行中/已结束）
.my-signup__list
├── .my-signup__card
└── .my-signup__empty
```

### 3.2 UI 元素清单

| 元素 | 文案 | 字段 |
|------|------|------|
| tab | 进行中(n)/已结束(n) | tabs |
| title | item.title | 活动名 |
| badge-active | **进行中** | listTab=active |
| badge-ended | **已结束** | listTab=ended |
| row-date | 📅 timeDisplay | |
| row-loc | 📍 location | wx:if location |
| meta | ¥fee · status · payStatus | 单行 |
| empty | 见空态表 | |

### 3.3 空态（精确）

| Tab | icon | title | hint |
|-----|------|-------|------|
| active | 📋 | **暂无进行中的报名** | **去首页看看热门招募，报名参加精彩水上活动** |
| ended | ✅ | **暂无已结束的报名** | **已结束的活动会显示在这里，方便查看历史记录** |
