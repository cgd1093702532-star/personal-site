## 2. 业务需求

### 2.1 业务目标

- 列表展示评价：评价者头像、昵称、星级、分数、内容、时间
- 个人中心展示 **已评价 N** 来自 `getMyReviewCount()`
- M1 **只读**，无发表/编辑评价入口

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 全部用户 | ✅ |

### 2.3 正常流程

进入 → 读 `reviews`（前台仅 `visible`）→ 列表展示；与后台同源。

### 2.4 核心业务规则

1. 读 `reviews`；前台仅展示 `visible`
2. 与后台评价数据同源
3. M1 只读，无发表/编辑入口
4. 星级支持半星；时间降序

### 2.5 异常与边界

- 无 avatar / content 时用占位或隐藏

### 2.6 待确认项

- [ ] 谁可评价
- [ ] 是否须先完成活动
- [ ] 提交入口是否全打通

## 3. 页面结构与 UI 元素规格

### 3.1 信息架构

```
.my-review__list
└── .my-review__item × N
    ├── head（avatar + name + stars + score）
    ├── content（可选）
    └── time（可选）
```

### 3.2 UI 元素清单

| 元素 ID | 类型 | 文案/内容 | 数据来源 |
|---------|------|-----------|----------|
| avatar | image/占位 | — | reviewer_avatar |
| name | 文本 | reviewer_nickname | 如 **学员小李** |
| star | 文本 | ★ ×5 | stars[].filled/half |
| score | 文本 | score 数字 | 如 5 / 4.5 |
| content | 文本 | 评价正文 | 如 **组织很好，体验棒！** |
| time | 文本 | timeDisplay | YYYY-MM-DD |
| empty | 文本 | **暂无评价** | list 空 |

#### 3.2.1 Mock 示例

| reviewer | score | content | time |
|----------|-------|---------|------|
| 学员小李 | 5 | 组织很好，体验棒！ | 2026-06-15 |
| 航海爱好者 | 4.5 | 教练很专业，场地设施也不错。 | 2026-06-08 |
| 张女士 | 5 | 孩子很喜欢，下次还来。 | 2026-05-28 |
