# 个人中心 · 商城版式对齐设计

> 日期：2026-07-14  
> 状态：已实现（预览 + 小程序 + 文档/截图）  
> 范围：微信小程序 Tab「我的」+ HTML 预览 + `docs/miniprogram/pages/个人中心.md`（含状态截图）

---

## 1. 目标

按商城个人中心参考图调整「我的」页版式与模块，同时保留已开发的英雄能力：

- **保留**：服务中心四入口及发布弹层；学员 / 评分 / 课程（改为独立「英雄数据」区块）
- **新增**：航海家权益卡；积分 / 赠品 / 优惠券·码；我的服务「购物车」
- **不做**：更多精选产品分区；顶部「请点击修改头像昵称」文案；消息铃铛

---

## 2. 已确认决策

| 项 | 结论 |
|----|------|
| 商城资产 vs 英雄统计 | 权益卡 + 积分/赠品/优惠券占原「头像下三统计」位；学员/评分/课程独立成「英雄数据」 |
| 英雄数据位置 | 我的订单之后、服务中心之前 |
| 购物车 / 精选产品 | 加购物车；不做精选产品 |
| 顶部信息 | 保留昵称 + 普通会员 + 认证标签；整体视觉贴参考图卡片风 |
| 可见性 | 已登录展示商城资产（未登录也可展示权益卡与三列占位，点击需登录的入口先授权）；英雄数据 / 服务中心仍仅「已认证且可用」 |

---

## 3. 页面结构（自上而下）

### 3.1 所有身份共有（或可展示）

1. **顶部信息**  
   头像 · 昵称（未登录「授权登录」）· 会员标签（已登录）· 认证标签（已登录）  
   去掉原嵌在用户头下的「学员/评分/课程」三列。

2. **航海家权益卡**  
   深蓝横幅；左侧「航海家权益卡」；右侧「立即开通 ›」。  
   本期：点击 → 未登录先授权；已登录可 toast「即将开放」或进占位（不接真实开通）。

3. **商城资产卡**  
   白卡片三列：积分 / 赠品 / 优惠券/码。  
   本期 mock 数字（默认 `0`）；点击 → 未登录先授权，已登录 toast「即将开放」。

4. **英雄申请 / 身份引导**（非「已认证可用」时）  
   文案与按钮逻辑**不变**（未登录 / 未申请 / 审核中 / 已驳回 / 禁用）。

5. **我的订单**  
   标题 +「全部订单 ›」+ 待付款 / 待发货 / 待使用·待收货 / 退款·售后。视觉贴参考图（白卡片、图标行）。

6. **我的服务**  
   列表行 + ›：  
   **购物车**（新增，置顶）→ 收货地址 → 在线客服 → 账号设置。  
   未登录点入口先授权；购物车本期可链商城或 toast「即将开放」。

### 3.2 仅已认证且可用

7. **英雄数据**（订单与服务中心之间）  
   三列：学员 / 评分 / 课程；数字与跳转逻辑与现网一致（我的学员、我的评分、我的课程）。

8. **服务中心**  
   发布招募/课程、我的招募、我的课程、修改资料（含审核中角标与弹层）。  
   保持四宫格快捷入口，不改为列表；外包白卡片以贴整体版式。

### 3.3 明确不做

- 「更多精选产品」分区及商品瀑布  
- 顶部改占位文案「请点击修改头像昵称」  
- 右侧消息铃铛  
- 积分/赠品/优惠券/权益卡真实业务与后端
