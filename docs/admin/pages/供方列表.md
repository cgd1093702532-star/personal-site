# 供方列表（管理后台）

> 状态：开发中 · P0 · 平台：Web 管理后台  
> UI 规范：[../UI-DESIGN-SPEC.md](../UI-DESIGN-SPEC.md)  
> 预览：`preview/admin/heroes.html` · 新建/编辑：`preview/admin/supplier-edit.html`

---

## 1. 页面概述

| 项 | 值 |
|---|---|
| 路由 | `/admin/heroes`（预览 `heroes.html`） |
| 新建/编辑 | `supplier-edit.html` / `supplier-edit.html?id=` |
| 权限 | 管理员（本地不鉴权） |
| 菜单位置 | 供需管理 → 供方列表 |

## 2. 列表页结构

1. **筛选区**（白底卡片，标签在上，对齐 UI 规范 tokens）
   - 供方 id、供方昵称/手机号码、项目类型、供方状态、审核状态
   - 操作：查询、重置、导出、查看已生成订单报表
2. **操作栏**：+ 创建供方（主按钮）、批量审核（描边主色）
3. **Tab**：全部 / 申请入驻待审核 (n) / 资料变更待审核 (n)
4. **表格列**：勾选、供方 id、供方信息（头像+昵称+手机）、项目类型、常驻城市、资质等级、从业年限、状态、审核状态、审核人员、审核时间、申请渠道、申请时间、操作

## 3. 操作

| 操作 | 行为 |
|------|------|
| 创建供方 | 跳转新建页（单列表单 + 底栏） |
| 详情（已入驻） | 跳转编辑页 |
| 详情 / 审核（待审申请） | 打开审核弹窗 |
| 批量审核 | 对勾选的待审申请打开审核 |

## 4. 新建 / 编辑页（UI 规范）

单列纵向表单，标签在上，必填星号在文案后，底栏固定：

| 字段 | 必填 | 控件 |
|------|------|------|
| 供方昵称 | 是 | Input，最多 20 字 |
| 手机号码 | 是 | Input |
| 供方头像 | 是 | Upload×1，建议 800×800 |
| 项目类型 | 是 | Input（顿号分隔） |
| 常驻城市 | 否 | Input |
| 资质等级 | 是 | Input |
| 从业年限 | 否 | Input |
| 备注 | 否 | Textarea |

底栏：取消（描边）+ 创建/保存（主色实心）

## 5. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/heroes` | 供方列表 |
| `POST /api/admin/heroes` | 创建供方 |
| `PUT /api/heroes/{id}` | 更新供方 |
| `GET /api/admin/applications` | 入驻申请列表 |
| `POST /api/admin/applications/{id}/approve\|reject` | 审核申请 |
| `GET /api/admin/profile-changes?status=pending` | 资料变更待审计数 |

## 6. 业务规则

1. 列表合并已入驻供方与待审/驳回申请；资料变更 Tab 依赖 profile-changes pending。
2. 后台创建渠道记为「后台创建」，审核状态默认「通过」。
3. 仅 `pending` 申请可审核。
4. 表单校验失败时高亮首个错误项并滚动定位。
