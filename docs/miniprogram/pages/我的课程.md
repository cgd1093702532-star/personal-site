## 1. 页面业务目标

「我的课程」是查看自己发起或参与的课程的地方。

主要解决两件事：

1. **按状态浏览**：进行中 / 已结束两个 Tab，看清哪些课还在招生、哪些已结束
2. **日常运营**：英雄可筛选「仅显示我发起的」、查看报名成员（仅「我发起的」）；卡片左上角区分「我发起的 / 我参与的」

当前版本可点卡片进入 [课程详情](./课程详情.md)。英雄也可从个人中心「发布招募/课程」→「申请课程」进入 [发布课程](./发布课程.md) 新增课程。

## 2. 页面详细描述

### 2.1 顶部 Tab

1、进行中 / 已结束，括号内为数量（如 `进行中(2)`、`已结束(1)`）。

2、仅显示我发起的：

2.1、仅英雄身份展示，非英雄不展示该筛选。

2.2、默认不勾选，勾选后当前 Tab 列表只保留「我发起的」数据，Tab 括号数量同步为筛选后条数；取消勾选恢复全部。

2.3、无论在哪个 Tab，勾选 / 取消勾选同时作用于进行中、已结束两个 Tab。

3、进行中取值和排序：

3.1、取进行中且已发布且未隐藏（若已发起课程，即便隐藏也显示）。

3.2、按开始时间正序、开始时间相同按结束时间正序，开始和结束都相同按创建时间倒序。

4、已结束取值和排序：

4.1、取已结束且已发布且未隐藏（若已发起课程，即便隐藏也显示）。

4.2、按结束时间倒序、结束时间相同按创建时间倒序。

### 2.2 课程卡片

| 元素 | 说明 |
|------|------|
| 我发起的/我参与的 | `我发起的`：本人发布，可看「已报名成员」<br>`我参与的`：他人发布且本人已报名成功，不展示「已报名成员」 |
| 左侧封面 | 封面图 |
| 标题 + 类型标签 | 标题；右上角「课程」标签 |
| 时间 | 与 [发布招募](./发布招募.md) 列表一致<br>1、取项目起止时间<br>2、同一天示例：`06/08 (周六) 09:00-16:00`<br>3、非同一天示例：`06/08 (周六) 09:00-06/09 16:00` |
| 地点 | `赛事地点：…` 或 `活动地点：…`（按类型） |
| 费用 | `¥X/人` |
| 底栏左 | `招募名额：已报/总额 · 状态`<br>进行中且未招满：`进行中`<br>进行中且已招满：`已招满`<br>已结束 Tab / 活动已结束：`已结束`<br>无上限：`…/不限 · 进行中` |
| 底栏右 | **我发起的**：进行中 / 已结束均为 `已报名成员`<br>**我参与的**：不展示该按钮 |

点卡片主体 → [课程详情](./课程详情.md)。点「已报名成员」→ [报名人员列表](./报名人员列表.md)（课程成员模式）。

### 2.3 空态

| Tab | 图标 | 标题 | 提示 |
|-----|------|------|------|
| 进行中 | 📚 | 暂无进行中的课程 | 可从个人中心「申请课程」发布 |
| 已结束 | ✅ | 暂无已结束的课程 | 已结束的课程会显示在这里，方便查看历史记录 |

> 空态无页内发布按钮；发布入口在个人中心「发布招募/课程」底部弹层。

## 3. 常见路径

- **查看课程：** 我的 → 我的课程 → 切换 Tab 浏览  
- **看详情：** 点课程卡片 → [课程详情](./课程详情.md)  
- **看成员：** 「我发起的」卡 →「已报名成员」→ [报名人员列表](./报名人员列表.md)  
- **我参与的：** 列表中「我参与的」卡可进详情，无「已报名成员」入口  
- **前台申请：** [个人中心](./个人中心.md) → 发布招募/课程 → 申请课程 → [发布课程](./发布课程.md) → 回到本页  
- **后台维护后回看：** 管理后台发布/编辑课程 → 本页刷新可见

## 5. 规则补充与验收要点

### 5.1 已对齐（产品已确认可验收）

- 两个 Tab（进行中 / 已结束）及括号内数量展示正确
- 英雄可筛选「仅显示我发起的」（默认不勾选；勾选后两 Tab 与括号数量同步）
- 卡片左上角关系：`我发起的` / `我参与的`；仅「我发起的」展示「已报名成员」
- 卡片布局对齐我的活动赛事：封面、标题/课程标签、时间、地点、费用、底栏招募名额
- 已结束 Tab 与进行中区分展示；无独立「已结束」角标、无进度条
- 空态文案符合 §2.3；发布入口在个人中心弹层
- 点击卡片进入课程详情
- 「已报名成员」进入成员列表（报名时间倒序：头像、昵称、手机号、时间）

- 与学员名册的关联跳转
- 是否需要「草稿」Tab，待产品确认
