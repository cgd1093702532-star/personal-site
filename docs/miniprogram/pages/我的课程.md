## 2. 业务需求

### 2.1 业务目标

- 展示教练发布的课程：进行中 / 已结束
- M1 **只读列表**，无编辑/下架/点进详情
- 与 [发布课程](./发布课程.md) 发布结果合并展示（storage + 默认 Mock）

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 已认证英雄 | ✅ |
| 其他 | ❌ |

### 2.3 正常流程

已认证进入 → 按当前英雄 `hero_id` 拉课程列表 → Tab 进行中 / 已结束。

### 2.4 核心业务规则

1. 仅已认证英雄可进入
2. 列表按 `hero_id` 过滤本人课程
3. Tab：进行中 / 已结束（可带计数）
4. 报名数据来自 `signups`（`type=course`），展示 signed/total
5. M1 只读列表（无编辑/下架入口）

### 2.5 异常与边界

- 空态无 CTA（与我的招募不同）
- onLoad / onShow 刷新

### 2.6 待确认项

- [ ] 课程草稿 / 上下架
- [ ] 结束 / 满员规则是否同招募

## 3. 页面结构与 UI 元素规格

### 3.1 信息架构

```
.my-courses__tabs
.my-courses__list
├── .my-courses__card × N
└── .my-courses__empty
```

### 3.2 UI 元素清单

| 元素 ID | 类型 | 文案 | 数据来源 | 交互 |
|---------|------|------|----------|------|
| tab | 文本 | **进行中(n)** / **已结束(n)** | tabs[] | onTabChange |
| title | 文本 | `{{item.title}}` | course | M1 无 tap |
| badge-ended | 标签 | **已结束** | listTab=ended | 无 |
| meta | 文本 | `{timeDisplay} · {location}` | course | 无 |
| price | 文本 | **¥{{item.price}}/人** | course | 无 |
| progress-label | 文本 | **已报名 {{signed}}/{{total}}** | active only | 无 |
| progress-bar | 条 | width=progress% | | 无 |
| empty-icon | 文本 | emptyState.icon | 📚/✅ | 无 |
| empty-title | 文本 | emptyState.title | 见下 | 无 |
| empty-hint | 文本 | emptyState.hint | 见下 | 无 |

### 3.3 空态（精确）

| Tab | icon | title | hint |
|-----|------|-------|------|
| active | 📚 | **暂无进行中的课程** | **发布课程，开始招募学员** |
| ended | ✅ | **暂无已结束的课程** | **已结束的课程会显示在这里，方便查看历史记录** |
