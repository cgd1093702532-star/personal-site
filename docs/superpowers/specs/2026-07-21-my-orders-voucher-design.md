# 我的订单 + 二维码凭证（设计）

> 日期：2026-07-21  
> 状态：已实现（含订单详情）  
> 相关：个人中心「我的订单」入口；参考图见会话附件（订单列表、二维码凭证、订单详情）

## 1. 目标

1. 个人中心可进入「我的订单」列表页（结构对齐参考图）。
2. 订单卡「券码凭证」进入「二维码凭证」页（版式完全对齐参考图）。
3. 点订单卡进入「订单详情」页（版式对齐参考图；演示数据用现有订单，不换商品）。
4. 预览与小程序同步；需求文档补充入口与新页说明。

## 2. 入口（个人中心）

| 入口 | 行为 |
|------|------|
| 「全部订单 ›」 | → 我的订单，Tab=`全部` |
| 「待付款」 | → 我的订单，Tab=`待付款` |
| 「待发货」 | → 我的订单，Tab=`待发货` |
| 「待使用/待收货」 | → 我的订单，Tab=`待使用/待收货` |
| 「退款/售后」 | → 我的订单，Tab=`全部`（本期不做退款 Tab） |

未登录：沿用个人中心现有授权门禁（与其它需登录入口一致）；已登录直接进入。

## 3. 我的订单页

### 3.1 结构

- 导航标题：我的订单；返回上一页。
- Tab（横向）：`全部`｜`待付款`｜`待发货`｜`待使用/待收货`｜`已完成`。
- 默认 Tab 由入口 query 决定（如 `?tab=pending_pay`）；非法/缺省 → `全部`。
- 列表为订单卡片；当前 Tab 无数据时居中空态（文案：`暂无相关订单`）。

### 3.2 订单卡（演示）

对齐参考图字段：

1. 顶栏：门店名（左，可带小图标）+ 状态文案（右，如「待使用」）。
2. 主体：封面缩略图 + 标题 + 单价 `¥X` + 数量 `xN`。
3. 汇总：右对齐 `实付款 ¥X`。
4. 操作：右下「券码凭证」（待使用）或「去评分」（已完成，进详情评分弹层）。

演示数据：至少 2 条——待使用「企业家杯月赛」¥5200；已完成「周末帆船体验课」¥680（无「券码凭证」）。「全部」可见两条；其它 Tab 按状态过滤（无数据则空态）。

### 3.3 交互

- 点订单卡主体 → 订单详情页（带订单 id）。
- 点「券码凭证」→ 二维码凭证页（带订单 id；不经详情）。
- Tab 切换仅过滤本地演示列表，不请求真实支付接口。

## 4. 二维码凭证页

版式完全按参考图：

1. 导航标题：二维码凭证。
2. 活动信息卡：标题；时间行（时钟图标 + 起止时间）；地点行（地点图标 + 地址 + 右侧 `>`）。
3. 凭证卡：提示「使用时请向工作人员出示二维码或券码」；二维码；券码文案；底部提示「为保障您的权益，未使用前请不要将二维码提供给商家」。
4. 底部「订单详情」卡：主文案「订单详情」、副文案「查看订单详情」、右侧 `>` → 订单详情页。

### 4.1 本期交互边界

| 元素 | 行为 |
|------|------|
| 地点行 `>` | 无反应（仅外观） |
| 「订单详情」卡 | → 订单详情页 |
| 二维码 / 券码 | 演示静态数据；已完成/已核销时二维码置灰并盖「已核销」印章 |

## 5. 订单详情页

版式按参考图；字段用演示订单（浪浪鲤 / 企业家杯月赛 / ¥5200），不换成参考图里的音乐会数据。

1. 顶栏状态区：状态文案（如「待使用」）。
2. 凭证信息卡：有效期；券码 +「复制」；右侧小二维码入口 → 二维码凭证页。
3. 门店 + 商品封面/标题/单价/数量 + 实付款。
4. 订单编号（可复制）、支付方式、创建时间、付款时间。
5. 底栏：待使用等为「取消报名」；已完成为「去评分」→ 底部「综合评分」弹层（5 星，未选星时提交置灰；提交成功 toast「评分成功」，页内展示「综合评分」+ 星星 + 分值，并隐藏「去评分」）。
6. 不做「更多精选产品」。
