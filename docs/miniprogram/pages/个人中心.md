> 状态：已确认 · P0 · M1  
> 最后更新：2026-07-06

---

## 1. 页面概述

| 项 | 值 |
|---|---|
| 路由 | `miniprogram/pages/profile/profile` |
| 页面标题 | **个人中心**（Tab 文案仍为「我的」） |
| 导航 | Tab 根页，**无返回按钮** |

## 2. 页面结构

### 2.1 顶部与用户信息

1. **导航栏** — 标题「个人中心」，无左上角返回
2. **用户信息区** — 头像、昵称；认证标签（未认证 / 已认证英雄）+ 会员级别（普通会员）；**已认证**时展示学员 / 评分 / 课程数据

### 2.2 英雄引导（双态）

| 状态 | 展示 |
|------|------|
| 未认证 | 文案左 +「申请成为英雄」按钮右 |
| 已认证 | 4 个快捷入口：发布招募/课程、我的招募、我的课程、英雄资料 |
| 审核中 | 审核提示条 |

### 2.3 我的活动

- **我的报名** — 图标 + 辅助文案（如「进行中 2，已完成 1」）
- **我的评价** — 图标 + 辅助文案（如「已评价 3」）

## 3. 功能说明

| 区块 | 说明 |
|------|------|
| 会员级别 | 示例「普通会员」，左侧并列认证标签 |
| 认证标签 | 「未认证」或「已认证英雄」 |
| 英雄引导 | 未认证：申请 CTA；已认证：三宫格快捷入口 |
| 发布招募 | 底部弹层：发布赛事招募 / 发布课程 / 取消 |
| 英雄资料 | → [修改英雄资料](./我的英雄资料.md)（子页右滑入，返回上一页） |

## 4. API

| 接口 | 用途 |
|------|------|
| `GET /api/users/me` | 用户信息 |
| `GET /api/heroes/me` | 英雄状态（null=未认证） |

## 4. 业务规则

- `heroes/me.status=pending` 时：显示「审核中」，隐藏英雄三宫格快捷入口
- `rejected` 时：允许重新 [申请成为英雄](./申请成为英雄.md)

## 5. 设计引用

- Tab 选中色：`--color-primary`
- 主按钮：`DESIGN-SPEC` §4.2 样式 A（金色 CTA）
