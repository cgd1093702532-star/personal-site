## 一、这份文档是干什么的？

你**不需要改代码**。以后想更新官网，只需：

1. **编辑本文件**里对应章节的内容
2. **保存**（Command + S）
3. 在 Cursor 对话里说：
  ```
   请根据 docs/网站内容需求文档.md 更新网站
  ```

AI 会读取本文档，自动同步到 `index.html` 和相关配置。

---

## 二、更新流程（标准操作）

```
修改本文档 → 保存 → 告诉 AI 同步 → 预览确认 → （可选）部署上线
```

### 2.1 预览网站

1. 双击项目里的 `启动预览.command`
2. 在中间 Browser Tab 粘贴：`http://127.0.0.1:5500/`
3. 改完内容同步后，浏览器 **Command + R** 刷新



### 2.2 告诉 AI 时可以这样说


| 场景    | 你可以说                      |
| ----- | ------------------------- |
| 全文同步  | 请根据 docs/网站内容需求文档.md 更新网站 |
| 只改一部分 | 请根据需求文档第 X 章更新网站          |
| 换了图片  | 图片已放好，请根据需求文档更新图片链接       |
| 新增案例  | 我在需求文档里加了案例 5，请更新网站       |




### 2.3 本文档 vs 其他文档


| 文档             | 你什么时候改它             |
| -------------- | ------------------- |
| **本文档**        | 改文案、价格、案例、联系方式、图片路径 |
| DESIGN-SPEC.md | 改颜色、字号、布局风格（一般不动）   |
| TECH-*.md      | 改技术方案（一般不动）         |

## 十八、文案写作规范（修改时请遵守）


| 规则  | 说明                         |
| --- | -------------------------- |
| 人称  | 多用「你」，少用「我们」               |
| 标题  | 具体场景 > 空泛形容词；避免「资深」「顶尖」    |
| CTA | 动词 + 低门槛，如「我想咨询」「立刻了解」     |
| 数据  | 数字 + 单位 + 标签；可用 `+` 表示「以上」 |
| 编号  | 全站两位数：01、02、03…            |
| 长度  | 卡片描述 2~3 行；单句不超过 28 字为宜    |
| 语气  | 专业、温暖、克制；知识型个人品牌           |
