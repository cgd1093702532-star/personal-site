# 评价管理（管理后台）

> 状态：已实现 · P1 · M2 · 平台：Web 管理后台

---

## 1. 功能

- **筛选：** 用户昵称/手机、供方昵称/手机、关联招募、关联项目、评价得分、评价状态
- **操作按钮：** 查询、重置、导出（占位）、查看已生成订单报表（占位）
- **表格：** 评价 id、用户 id、用户信息、供方 id、供方信息、关联招募、关联内容、关联订单、评价得分、评价内容、评价状态、提交时间、操作
- **操作：** 隐藏（公开 → 隐藏）
- **风格：** 与供方列表一致（`admin-filter-panel` + 绿表头表格）

## 2. 页面结构

1. 筛选面板（无状态 Tab）
2. 评价列表表
3. 行内隐藏

## 3. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/reviews` | 后台看全部评价（前端过滤 deleted） |
| `GET /api/admin/heroes` | 合并供方信息 |
| `GET /api/admin/users` | 合并用户手机等 |
| `POST /api/admin/reviews/{review_id}/hide` | 隐藏 → `hidden` |

## 4. 说明

评价正式数据在 SQLite / API。列表展示字段可来自 review payload，缺省时由英雄/用户表补齐。

状态文案：`visible` → 公开，`hidden` → 隐藏。

## 5. 正常流程

打开评价列表 → 筛选 → 隐藏不当评价。

## 6. 业务规则

1. 后台默认展示公开与隐藏，不含已删除。
2. 隐藏将状态置为 `hidden`。
3. 导出 / 订单报表暂为占位提示。

## 7. 异常与边界

- 关联字段缺失显示 `—` / `暂无`。
- 本地 API 未启动时页面提示启动命令。
