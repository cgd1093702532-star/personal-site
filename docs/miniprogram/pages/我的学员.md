## 2. 业务需求

### 2.1 业务目标

- 展示教练关联学员列表，支撑个人中心「学员」统计入口
- M1 只读 Mock/API app-state，无搜索、无详情页

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 已认证英雄 | ✅ |
| 其他 | ❌（profile onMyStudents 守卫） |

### 2.3 正常流程

已认证进入 → 由该英雄招募/课程的 `signups` 聚合学员列表。

### 2.4 核心业务规则

1. 学员来自该英雄招募/课程的 `signups` 聚合
2. 取消报名不计入（实现：`status != cancelled`）
3. M1 只读，无搜索/详情
4. 可按 `last_active` 降序

### 2.5 异常与边界

- 非英雄无入口

### 2.6 待确认项

- [ ] 学员去重键（`user_id` / `phone`）

## 3. 页面结构与 UI 元素规格

### 3.1 信息架构

```
.my-student__list
├── .my-student__item × N
│   ├── avatar
│   └── info（name + meta）
└── .my-student__empty
```

### 3.2 UI 元素清单

| 元素 ID | 类型 | 文案/格式 | 数据来源 | 交互 |
|---------|------|-----------|----------|------|
| avatar | image/占位 | — | item.avatar | 无 |
| name | 文本 | `{{item.nickname}}` | list | 无 |
| meta | 文本 | **参与 {{course_count}} 门课程 · 最近活跃 {{last_active}}** | list | 无 |
| empty | 文本 | **暂无学员** | list.length=0 | 无 |
