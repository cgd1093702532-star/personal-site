# 系统配置（管理后台）

> 状态：草稿 · P2 · M3 · 平台：Web 管理后台

---

## 1. 配置项

### 业务场景

- 场景列表：水上运动、酒店预订、场地预约等
- 增删改停用；字段：名称、编码、状态

### 概念命名（文案）

- 「英雄」「招募」「报名」等文案替换（`copy`）

### 功能开关（`features`）

- 英雄认证 / 评级 / 证书展示 / 在线支付（预留）/ 地理定位

## 2. 页面结构

1. 场景（scenarios）管理
2. 文案（copy）配置
3. 功能开关（features）

## 3. API

| 接口 | 用途 |
|------|------|
| `GET /api/admin/scenarios` | 场景列表 |
| `PUT /api/admin/scenarios/{scenario_id}` | 更新场景（含启停） |
| （规划）copy / features 读写 | 文案与功能开关 |

## 4. 说明

配置项分组：`scenarios` / `copy` / `features`。本地不做真实支付。

## 5. 正常流程

打开系统配置 → 维护场景启停 / 文案 / 功能开关 → 保存。

## 6. 业务规则

1. 配置分 `scenarios`、`copy`、`features` 三类。
2. 场景支持启用 / 停用。
3. 功能开关含：认证、评级、证书、支付、定位。

## 7. 异常与边界

- 停用场景后，依赖该场景的列表筛选应不再展示或标记不可用。
- 开关关闭时的前台行为见待确认（藏入口 vs 仅禁用提交）。
