## 2. 业务需求

### 2.1 业务目标

- 管理本人发布的招募：草稿、进行中、已结束三 Tab
- 进行中：查看报名、编辑、下架
- 已截止/结束：查看名单、数据（M1 Toast 开发中）
- 草稿：继续编辑、删除
- 顶部月统计：本月报名人次、本月收入（M1 Mock 静态）

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 已认证英雄 | ✅ |
| 其他 | ❌（入口守卫） |

### 2.3 正常流程

已认证进入 → 拉本人招募列表 → Tab：进行中 / 已结束 / 草稿 → 编辑、下架、删除、查看报名。

### 2.4 核心业务规则

1. 仅已认证英雄可进入
2. Tab：`active` / `ended` / `draft`（文案可带计数）
3. 可删除草稿；进行中可下架（归 ended）
4. 后台关闭 → `cancelled`（归已结束侧）
5. 非草稿卡片可进招募详情；草稿走继续编辑

### 2.5 异常与边界

- 列表空态按 Tab 展示不同文案
- onLoad / onShow 均刷新
