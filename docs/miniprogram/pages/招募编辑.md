## 2. 业务需求

### 2.1 业务目标

- 编辑已有招募/草稿：表单与 `recruitment-form.js` 完全共享
- **保存草稿**：保持 displayStatus=draft, listTab=draft
- **保存修改**：全量校验 → resolvePublishedStatus → updateRecruitment → navigateBack

### 2.2 适用角色与权限

| 角色 | 可否进入 |
|------|----------|
| 发布者（英雄） | ✅ |
| id 缺失/不存在 | ❌ Toast |

### 2.3 核心业务规则

1. `options.id` 缺失 → Toast **缺少招募 ID**
2. `getRecruitmentById` 不存在 → **招募不存在**
3. `itemToForm` 预填；保留 sourceItem 用于 merge recruit_id/signed 等
4. 草稿保存：updateRecruitment + storage draft；Toast **草稿已保存**（不 redirect）
5. 保存修改：removeStorage draft；Toast **保存成功** → 800ms navigateBack
6. 字段/校验/地图/封面/audience 与发布页一致
