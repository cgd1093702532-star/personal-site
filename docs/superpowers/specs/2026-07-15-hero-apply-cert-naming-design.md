# 申请成为英雄 · 资质证书命名交互

> 日期：2026-07-15  
> 状态：已实现

## 目标

上传资质证书后必须为每张图片命名；交互符合移动端填表习惯。

## 约定

| 项 | 规则 |
|----|------|
| 选图 | 每次只选 1 张（关闭多选） |
| 命名时机 | 选图后立即弹窗命名；进列表后仍可在行内修改 |
| 名称 | 必填，最多 15 字 |
| 上限 | 最多 10 张；满 10 张隐藏添加入口 |
| 列表形态 | 竖排证书条（方案 B） |
| 取消弹窗 | 丢弃本次选图，不进列表 |

## 上传弹窗

- 缩略图预览 + 名称输入（placeholder「请输入证书名称」）
- 按钮：取消 / 确认添加
- 空名确认 → toast「请填写证书名称」

## 列表

每行：左侧缩略图、右侧名称输入、删除；底部全宽「上传证书 / 添加更多」。

## 提交

- 至少 1 张证书（沿用现规则）
- 任一名称为空 →「请填写证书名称」
- 提交数据结构：`certificates: [{ name, image }]`，并写 `cert_count`
